Java中的重载和重构

对于一个像小郭一样的新人，也就是本人。几乎每次都要在重载(overload)和重写(override)上面费下脑子，时不时的看着屏幕上的 @overload伤脑筋，到底是什么意思？
下来就来分别讲下他们的区别:

重写(override)

重写的重点就在一句： 同名同参同返不同体 就是相同的名字（外壳），相同的参数，相同的返回类型，但不同的核心。

重写一般存在于继承关系中，是子类对父类的允许访问的方法的实现过程进行重新编写，返回值和形参都不能改变。

重写的规则

  1. 子类只能重写父类非 final 修饰的代码，换句话说如果父类的代码为final，即表示其不可被重写。
  2. 访问权限不能比父类中被重写的方法的访问权限更低。例如：如果父类的一个方法被声明为 public，那么在子类中重写该方法就不能声明为 protected。
  3. 重写的原因在于子类可以根据自己的需要，定义属于自己的行为通过实现父类同名的方法。
  4. 声明为 static 的方法不能被重写，但是能够被再次声明。
  5. 参数列表必须完全与被重写方法的相同。

重载(overload)

重载是指一个类中，方法名字相同，但是参数不同，返回类型可相同也可不同。

每个重载的方法都必须有一个独一无二的参数类型列表。

常用于构造器的重载，如果一个类有多个构造函数，Java会基于所用的参数类型来选择最合适的那一个

重载

  1. 被重载的方法必须改变参数列表(参数个数或类型不一样)；

  2. 被重载的方法可以改变返回类型；

  3. 被重载的方法可以改变访问修饰符；

  4. 被重载的方法可以声明新的或更广的检查异常；

  5. 方法能够在同一个类中或者在一个子类中被重载。

  6. 无法以返回值类型作为重载函数的区分标准。

小结

  1. 重写（override）就是在已有方法的基础上，不改变其参数，返回值，但内部处理的核心不一样。
  2. 重载（overloading）是必须要修改参数类型，可以修改返回类型，例如让不同的构造器产生不同的对象。
